# Graph Transformer 模型精度修复计划

## 背景

审查发现 Graph Transformer 模型存在多个影响精度的问题，其中最严重的是 GCN 层边特征退化为标量门控。本计划分阶段修复，优先解决影响精度的核心问题。

## 修复内容

### 1. [关键] 修复 GCN 层边特征传递 — 无界逐元素改为有界门控调制
- **文件**: `graph_transform/models/gcn_layers.py:97-118`
- **当前状态**: 已从标量门控改为逐元素 `x[src] * edge_attr * norm`，但 edge_attr 经纯 Linear 后无界，训练后期存在数值爆炸风险
- **修复**: 添加 `sigmoid(edge_attr)` 逐维有界门控 + 门控加权度归一化
- **具体改动**:
  ```python
  # propagate_with_edges 中：
  edge_gate = torch.sigmoid(edge_attr)                  # [E, 256] 有界门控
  deg.index_add_(0, col, edge_gate.mean(dim=1))         # 门控加权度（非计数度）
  messages = x_compute[row] * edge_gate * norm.unsqueeze(1)
  ```
- **效果**: 256 维边特征差异化调制节点消息 + sigmoid 保证数值稳定

### 2. [建议] 关闭长程边作为稳定 baseline
- **文件**: `graph_transform/config/default.yaml:41`
- **修改**: `use_long_range_edges: true` → `use_long_range_edges: false`
- **原因**: 对 ≤32 长度的肽段，stride=10 的长程边与 distance graph 高度重叠，无额外信息但增加计算开销

### 3. [建议] 修复 LazyLinear 初始化不一致
- **文件**: `graph_transform/models/graph_transformer.py:314`
- **问题**: `edge_attr_encoder = nn.LazyLinear(...)` 使用默认随机初始化，而非与其它层一致的 Xavier
- **修复**: 替换为普通 `nn.Linear`，根据已知 edge_attr 维度（8，来自 `_create_edge_features`）指定 input_dim

### 4. [建议] 物理化学特征归一化
- **文件**: `graph_transform/models/graph_transformer.py:246-253`
- **问题**: 4 个物化属性尺度差异大（分子量 75-204 vs 电荷 -1~1）
- **修复**: 在 lookup table 构建时进行 min-max 归一化到 [0, 1]

### 5. [低优先] 清理未使用的 layer_norms
- **文件**: `graph_transform/models/graph_transformer.py:395-397`
- **说明**: 这些 LayerNorm 参数不参与前向计算，占用显存，但保留用于 checkpoint 兼容性

### 6. [低优先] 修复 get_attention_weights 的 global node 兼容
- **文件**: `graph_transform/models/graph_transformer.py:629-661`
- **说明**: 仅影响可视化，非功能性 bug

## 不修改的内容
- `max_distance: 10` 保持不变（对 ≤32 肽段已足够）
- `long_range_stride/hops` 参数保留配置，仅关闭开关
- `SequenceGraphBuilder.build_advanced_graph` 死代码暂不清理（非本次重点）

## 修改文件清单
1. `graph_transform/models/gcn_layers.py` — 修复 propagate_with_edges
2. `graph_transform/config/default.yaml` — 关闭 use_long_range_edges
3. `graph_transform/models/graph_transformer.py` — LazyLinear → Linear + 物化归一化

## Global Node 分析结论

Global node 设计合理，图构建正确（双向连边 + state/env 编码初始化）。当前经过 GCN 层时 edge_attr 无界，global node 的 256 维信息在传递时缺乏稳定门控。修复 GCN 的有界门控后，global node 作为"全链信息汇聚点"的作用会显著增强。

## 验证方式（模型修复）
- 修改后运行训练脚本确认无运行时错误
- 对比修改前后的训练 loss 收敛曲线
- 重点关注 GCN 修复后 loss 是否更快下降

## 5 折交叉验证训练脚本

### 背景

五折数据集位于 `dataset/5fold/`，包含 5 组 train/test 对：
- `{fold_id}.train.fbr.shuffle.multi.csv` / `{fold_id}.test.fbr.multi.csv`
- fold_id: `1222, 2252, 3514, 6072, 9075`

原先训练命令 `python graph_transform/scripts/train_graph_model.py --config graph_transform/config/default.yaml`，数据路径写死在 yaml 的 `data.train_csv_path` / `data.test_csv_path` 中。

### 新建文件

**`graph_transform/scripts/train_5fold.py`**

### 设计要点

1. **CLI 接口**
   ```bash
   python graph_transform/scripts/train_5fold.py \
       --config graph_transform/config/default.yaml \
       --fold_dir dataset/5fold \
       [--folds 1222,2252,3514,6072,9075]  \   # 可选，默认自动发现
       [--epochs 60] \
       [--batch_size 1024] \
       [--seed 42] \
       [--device cuda]
   ```

2. **自动发现折次**
   - 扫描 `--fold_dir` 下的 `*.train.fbr.shuffle.multi.csv` 文件
   - 从文件名提取 fold_id（如 `1222`）
   - `--folds` 参数可覆盖，指定子集或不同顺序

3. **每折执行流程**（复用 `train_graph_model.py` 的核心函数）

   ```
   for fold_id in folds:
       1. 深拷贝 config
       2. 覆盖 config['data']['train_csv_path'] = fold_dir/{fold_id}.train.fbr.shuffle.multi.csv
       3. 覆盖 config['data']['test_csv_path']  = fold_dir/{fold_id}.test.fbr.multi.csv
       4. 覆盖 config['data']['val_csv_path'] = null（保持原逻辑：从训练集按 validation_split=0.2 分割出 val set）
       5. 覆盖 config['training']['checkpoint_dir'] = checkpoints/graph_transform/5fold/fold_{fold_id}/
       6. 覆盖 config['logging']['log_dir'] = logs/graph_transform/5fold/fold_{fold_id}/
       7. 设置独立随机种子: seed + fold_index
       8. 重新创建 model / optimizer / scheduler / trainer / evaluator（每折独立）
       9. 执行完整训练循环（与 train_graph_model.py main() 中相同的逻辑）
      10. 训练结束后加载 best_model.pt，在 test set 上评估
      11. 保存 per-fold 结果: metrics csv + pred csv + training history
   ```

4. **结果汇总**

   训练完所有折后：
   - 收集每折的 test metrics（loss, accuracy, precision, recall, f1, auc）
   - 计算 mean ± std
   - 输出到 `result/metric/graph_transform/5fold/5fold_summary.csv`
   - 格式：
     ```
     fold_id, loss, accuracy, precision, recall, f1, auc
     1222,   0.xxx, 0.xxx,   0.xxx,    0.xxx,  0.xxx, 0.xxx
     2252,   0.xxx, 0.xxx,   0.xxx,    0.xxx,  0.xxx, 0.xxx
     ...
     mean,   0.xxx, 0.xxx,   0.xxx,    0.xxx,  0.xxx, 0.xxx
     std,    0.xxx, 0.xxx,   0.xxx,    0.xxx,  0.xxx, 0.xxx
     ```
   - 打印汇总表格到日志

5. **代码复用策略**

   从 `train_graph_model.py` 导入以下函数，不复制粘贴：
   - `setup_logging`, `setup_device`
   - `create_model`, `create_optimizer`, `create_scheduler`
   - `create_datasets`, `create_data_loaders`
   - `save_training_curves`, `save_evaluation_outputs`
   - `log_metrics_to_tensorboard`, `build_evaluation_id`
   - `load_config`, `create_tensorboard_writers`
   - `flush_tensorboard_writers`, `close_tensorboard_writers`

   新脚本中仅实现：
   - `discover_folds(fold_dir)` — 自动发现折次
   - `run_single_fold(config, fold_id, fold_dir, device, ...)` — 单折训练
   - `aggregate_fold_results(all_fold_metrics)` — 汇总统计
   - `main()` — 参数解析 + 循环调度

6. **每折独立隔离**

   - 每折创建全新的 model、optimizer、scheduler（不跨折共享参数）
   - 每折独立的 checkpoint 目录、log 目录、TensorBoard 目录
   - 每折训练结束后释放 GPU 显存（`torch.cuda.empty_cache()`）
   - 每折使用不同随机种子：`seed + fold_index`

7. **错误处理**

   - 单折失败不中断整体流程，记录错误并继续
   - 最终汇总报告中标记失败折次
   - 支持 `--resume_fold fold_id` 从某一折继续
