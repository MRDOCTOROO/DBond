# 代码优化计划：用库函数替换自定义实现 & 算法优化

## 概述

对整个 DBond 项目（graph_transform、mini_ghtrans、arc_dbond、PBCLA）进行全面审查后，发现大量自定义函数可被现有 Python 库替换，以及多个算法层面的性能瓶颈。以下是按优先级排序的优化建议。

---

## P0 - 关键性能瓶颈（训练速度影响最大）

### 1. 自定义分段 softmax → `torch_geometric.utils.softmax`

**文件**: `graph_transform/models/attention_layers.py:218-242`, `mini_ghtrans/models/attention_layers.py:128-141`

当前手动实现 scatter softmax（`scatter_reduce_("amax")` + `exp()` + `scatter_add_()` + 除法），每个 forward pass 都在注意力层中调用。

**替换为**:
```python
from torch_geometric.utils import softmax
attention_weights = softmax(e, target_index, num_nodes=num_nodes)
```
**预期加速**: 2-4x 注意力计算（CUDA 优化内核）

### 2. 手动 scaled dot-product attention → `F.scaled_dot_product_attention`

**文件**: `mini_ghtrans/models/attention_layers.py:239-257`

手动实现 Q·K^T/sqrt(d) + mask + softmax + dropout + Q·V。PyTorch 2.0+ 的 `F.scaled_dot_product_attention` 使用 Flash Attention，内存从 O(N^2) 降到 O(N)。

**预期加速**: 2-4x，内存大幅降低

### 3. Python 循环遍历每条边做 NN forward → 完全向量化

**文件**: `mini_ghtrans/models/gcn_layers.py:392-424` (`EdgeConditionedConvLayer.forward`)

```python
for i in range(edge_index.size(1)):
    edge_input = torch.cat([x[src], edge_attr[i]], dim=-1)
    core_out = self.core_net(edge_input)   # 每条边单独调用NN
```

**替换为**:
```python
edge_input = torch.cat([x[row], edge_attr], dim=-1)
core_out = self.core_net(edge_input)
gate_out = self.gate_net(edge_input)
messages = gate_out * core_out
output = torch.zeros_like(x)
output.index_add_(0, col, messages)
```
**预期加速**: 100-1000x（消除 Python 循环 + 批量化 NN）

### 4. Python 循环遍历节点做 max 聚合 → `scatter_reduce_("amax")`

**文件**: `mini_ghtrans/models/gcn_layers.py:259-273` (`MaxAggregator.forward`)

```python
for i in range(x.size(0)):
    neighbors = row == i
    aggregated[i] = torch.max(x[neighbors], dim=0)[0]
```

**替换为**: `torch.zeros_like(x).scatter_reduce_(0, col, x[row], reduce="amax")`
**预期加速**: 100x+

### 5. 缺少 `torch.no_grad()` 上下文 — 评估时仍构建计算图

**文件**: `arc_dbond/train.dbond_s.py:99-156`, `arc_dbond/train.dbond_m.py:99-155`, `arc_dbond/evaluate.dbond_s.py:192-284`, `arc_dbond/evaluate.dbond_m.py:131-224`

评估阶段仅调用 `model.eval()` 但未用 `torch.no_grad()`，导致 GPU 额外分配内存存储梯度信息。

**修复**: 在 eval 模式下用 `with torch.no_grad():` 包裹前向传播和 loss 计算。

## P1 - 重要替换（库函数替换自定义实现）

### 6. 自定义 GELU/SiLU/Mish → PyTorch 内置

**文件**: `graph_transform/models/utils.py:187-199`, `mini_ghtrans/models/utils.py:160-172`

```python
def gelu(x):  return 0.5 * x * (1.0 + torch.erf(x / math.sqrt(2.0)))   # → F.gelu(x)
def swish(x): return x * torch.sigmoid(x)                                # → F.silu(x)
def mish(x):  return x * torch.tanh(F.softplus(x))                       # → F.mish(x)
```

PyTorch 内置版本有 CUDA 优化内核。`leaky_relu_custom` 纯粹是 `F.leaky_relu` 的无用包装，直接删除。

### 7. 手动 GCN 归一化 → `torch_geometric.nn.GCNConv` 或 `gcn_norm`

**文件**: `graph_transform/models/gcn_layers.py:82-95`, `mini_ghtrans/models/gcn_layers.py:77-94`

手动实现 D^{-1/2} A D^{-1/2} 归一化 + scatter message passing。

**替换为**: `torch_geometric.utils.gcn_norm(edge_index)` + PyG 的 GCNConv（CUDA 融合内核）。
**预期加速**: 2-5x message passing

### 8. 自定义指标计算 → `sklearn.metrics` 直接调用

**文件**: `arc_dbond/multi_label_metrics.py` 全文

以下自定义函数均可被一行 sklearn 调用替换：

| 自定义函数 | 行号 | sklearn 替换 |
|---|---|---|
| `label_precision_macro` | 64-68 | `precision_score(gt, pred, average='macro')` |
| `label_precision_micro` | 71-74 | `precision_score(gt, pred, average='micro')` |
| `label_recall_macro` | 77-81 | `recall_score(gt, pred, average='macro')` |
| `label_recall_micro` | 84-87 | `recall_score(gt, pred, average='micro')` |
| `label_f1_macro` | 90-95 | `f1_score(gt, pred, average='macro')` |
| `label_f1_micro` | 98-103 | `f1_score(gt, pred, average='micro')` |
| `example_subset_accuracy` | 11-13 | `accuracy_score(gt, pred)` |
| `example_accuracy` | 16-19 | `jaccard_score(gt, pred, average='samples')` |
| `_label_quantity` | 42-47 | `multilabel_confusion_matrix(gt, pred)` |

同样适用于 `graph_transform/training/metrics.py:261-296` 中冗余的 micro/macro/weighted 指标计算（对二分类它们值相同却调用了3次）。

### 9. O(n^2) Python 双重循环 → numpy 向量化距离矩阵

**文件**: `graph_transform/data/graph_builder.py:293-309`, `mini_ghtrans/data/graph_builder.py:246-262`

```python
for i in range(seq_len):
    for j in range(seq_len):
        distance_matrix[i, j] = abs(i-j) * (1.0 + 0.2*np.sin(abs(i-j)*0.5))
```

**替换为**:
```python
idx = np.arange(seq_len)
seq_dist = np.abs(idx[:, None] - idx[None, :])
distance_matrix = seq_dist * (1.0 + 0.2 * np.sin(seq_dist * 0.5))
np.fill_diagonal(distance_matrix, 0)
```
**预期加速**: 50-200x

### 10. 顺序遍历 50 个阈值搜索最优 F1 → 向量化

**文件**: `graph_transform/training/metrics.py:308-318`, `mini_ghtrans/training/metrics.py:140-154`

当前对 50 个阈值循环调用 `f1_score` 50 次。

**替换为** numpy broadcasting 一次性计算所有阈值的 TP/FP/FN，然后算 F1。
**预期加速**: ~50x

### 11. 稠密邻接矩阵 → 稀疏操作

**文件**: `graph_transform/models/utils.py:258-300`, `mini_ghtrans/models/utils.py:231-273`, `mini_ghtrans/models/gcn_layers.py:347-363`

构建 N×N 稠密矩阵 + 两次矩阵乘法做归一化。

**替换为**: `torch_geometric.utils.gcn_norm(edge_index)` 直接在稀疏 edge_index 上操作。
**影响**: 10-100x 内存降低，消除 O(n^3) 矩阵乘法

### 12. Python 循环提取子图 → `torch_geometric.utils.subgraph`

**文件**: `graph_transform/models/utils.py:302-327`, `mini_ghtrans/models/utils.py:276-300`

```python
for i in range(edge_index.size(1)):
    src, dst = edge_index[0, i], edge_index[1, i]
    if src.item() in node_map and dst.item() in node_map: ...
```

`.item()` 在 GPU 上触发 CUDA 同步。**替换为**: `torch_geometric.utils.subgraph(nodes, edge_index)`

## P2 - 中等优先级（代码质量 + 中等性能提升）

### 13. L2 正则化 → optimizer `weight_decay`

**文件**: `graph_transform/models/utils.py:237-242`, `mini_ghtrans/models/utils.py:209-215`

手动遍历所有参数计算 `param**2` 之和。AdamW 的 `weight_decay` 参数已处理此逻辑，无需额外计算图节点。

### 14. 梯度范数双重计算

**文件**: `graph_transform/training/trainer.py:544-555`

`_compute_grad_norm` 手动计算梯度范数，但 `_backward_pass` 中又调用 `clip_grad_norm_` 重新计算。`clip_grad_norm_` 的返回值就是总范数，直接使用即可。

### 15. collate 函数中 `.item()` 导致 CUDA 同步

**文件**: `graph_transform/data/graph_dataset.py:296-324`, `graph_transform/data/optimized_graph_dataset.py:262-352`

```python
batch_edge_index = [item['edge_index'] + int(offset.item()) for ...]
```

`int(offset.item())` 每个样本触发一次 CUDA 同步。改用 tensor 算术运算。

### 16. numpy→torch→sigmoid→numpy 转换 → `scipy.special.expit`

**文件**: `graph_transform/training/metrics.py:86-91`, `mini_ghtrans/training/metrics.py:62-63`

```python
torch.sigmoid(torch.from_numpy(values)).numpy()  # → scipy.special.expit(values)
```

### 17. 字符编码双重 Python 循环 → 预计算查找表

**文件**: `mini_ghtrans/models/graph_transformer.py:140-157`

逐字符 `char_to_idx[char]` 查找。可预计算 numpy lookup array 实现向量化。

### 18. 手动欧式距离 → `scipy.spatial.distance.euclidean`

**文件**: `graph_transform/data/graph_builder.py:447-460`, `mini_ghtrans/data/graph_builder.py:395-408`

### 19. O(N*M) pandas 行级过滤 → `DataFrame.merge`

**文件**: `arc_dbond/evaluate.dbond_s.py:312-325`

逐行扫描 DataFrame 做多列条件过滤。单次 `pd.merge(on=condition_col)` 即可完成。

### 20. 108 行自定义浮点二分搜索 → `bisect` 模块

**文件**: `PBCLA/utils.py:90-197`

可用 5-10 行 `bisect.bisect_left` + `bisect.bisect_right` 替换。

### 21. PBCLA 键匹配 O(bonds×ions) → O(ions) 哈希表

**文件**: `PBCLA/pbcla.py:63-82`

预建 `bond_lookup` 字典后单次遍历 matched_ions。

## 实施建议

建议按以下顺序执行：

1. **第一阶段** (训练速度提升最大): P0 的 #1-#5
2. **第二阶段** (库函数替换): P1 的 #6-#12
3. **第三阶段** (代码质量): P2 的 #13-#21 + P3 的 #22-#26
4. **第四阶段** (Bug 修复): #27-#28

预计总体训练速度提升 **2-5x**（主要来自向量化注意力、GCN、Flash Attention 和 `torch.no_grad()` 修复），代码量减少 **~600+ 行**（消除重复代码）。
